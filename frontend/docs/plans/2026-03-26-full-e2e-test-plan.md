## 测试模板设计

### 模板名称：综合素质量化评估
### 模板结构

```
根分区: 综合素质量化评估 (targetType: ORG, targetFilter: unit_type=CLASS)
├── 分区1: 卫生检查 (weight: 30, targetType: ORG, inputMode: INLINE)
│   ├── 地面清洁 (PASS_FAIL) passScore:10 failScore:0
│   │   └── 验证: 必填
│   ├── 桌椅摆放 (DEDUCTION) maxScore:10 deductPerViolation:2 step:2
│   │   └── 验证: 必填
│   ├── 窗台灰尘 (LEVEL) 优10/良7/中4/差0
│   │   └── 验证: 必填
│   ├── 黑板清洁 (RATING_SCALE) maxRating:5 scorePerStar:4 maxScore:20
│   │   └── 验证: 必填
│   └── 垃圾分类 (ADDITION) maxBonus:10 step:5 items:[分类正确+5, 桶内无溢出+5]
│       └── 条件: 仅当"地面清洁"=通过时显示
│
├── 分区2: 纪律检查 (weight: 30, targetType: ORG, inputMode: INLINE)
│   ├── 出勤率 (DIRECT) minScore:0 maxScore:20
│   │   └── 验证: 必填, 范围0-20
│   ├── 迟到人数 (CUMULATIVE) scorePerItem:-3 countLabel:人
│   │   └── 验证: 必填
│   ├── 课堂秩序 (TIERED_DEDUCTION) tiers:[无违纪0, 1-2次-5, 3-5次-10, 5次以上-20]
│   │   └── 验证: 必填
│   ├── 违纪记录 (VIOLATION_RECORD, inputMode: EVENT_STREAM) deductPerViolation:5
│   │   └── 录入模式: 巡查快速记录(搜学生记录)
│   └── 纪律综合评级 (SCORE_TABLE) table:[A优秀20, B良好15, C合格10, D不合格0]
│       └── 条件: 仅当所有前置项都已填写时显示
│
├── 分区3: 文明行为 (weight: 20, targetType: ORG, inputMode: INLINE)
│   ├── 着装规范 (PASS_FAIL) passScore:15 failScore:0
│   │   └── 需要证据(拍照)
│   ├── 礼貌用语 (LEVEL) 优秀15/良好10/一般5/差0
│   └── 公物爱护 (THRESHOLD) thresholds:[0处损坏=20, 1-2处=15, 3-5处=8, 6+处=0]
│       └── 验证: 必填, 数值>=0
│
└── 分区4: 学习表现 (weight: 20, targetType: ORG, inputMode: INLINE)
    ├── 作业完成率 (DIRECT) minScore:0 maxScore:20
    │   └── 验证: 必填
    ├── 课堂参与度 (WEIGHTED_MULTI) items:[积极发言w0.3, 认真听讲w0.3, 按时交作业w0.2, 课堂笔记w0.2] maxScore:20
    └── 学习综合 (FORMULA) formula:score maxScore:20
        └── 描述: 根据整体表现综合打分
```

## 测试步骤

### Phase 1: 管理员 — 模板创建
1. 登录系统 (admin/admin123)
2. 进入 检查配置 页面
3. 点击 "新建模板"
4. 创建根分区: 综合素质量化评估
5. 添加4个子分区 (卫生/纪律/文明/学习)
6. 每个分区添加检查项 (覆盖所有评分模式)
7. 配置评分参数
8. 设置等级映射
9. 设置条件逻辑 (垃圾分类依赖地面清洁)
10. 设置证据要求 (着装规范需拍照)
11. 预览模板
12. 发布模板

### Phase 2: 管理员 — 项目创建
13. 进入 检查项目 页面
14. 点击 "新建项目"
15. 选择模板: 综合素质量化评估
16. 配置范围: 选5个班级
17. 配置日期: 今天开始,一周后结束
18. 添加检查员: admin
19. 发布项目

### Phase 3: 检查员 — 我的检查
20. 进入 "我的检查" 页面
21. 查看今日待办
22. 点击任务进入打分界面

### Phase 4: 检查员 — 打分
23. 选择第一个目标 (班级)
24. 逐项打分:
    - 通过/不通过 (PASS_FAIL)
    - 扣分步进器 (DEDUCTION)
    - 等级选择 (LEVEL)
    - 星级评分 (RATING_SCALE)
    - 加分项 (ADDITION)
    - 直接打分 (DIRECT)
    - 计次 (CUMULATIVE)
    - 分档扣分 (TIERED_DEDUCTION)
    - 快速记录 (EVENT_STREAM/VIOLATION_RECORD)
    - 评分标准表 (SCORE_TABLE)
    - 阈值判定 (THRESHOLD)
    - 多维加权 (WEIGHTED_MULTI)
    - 公式计算 (FORMULA)
25. 验证条件逻辑 (垃圾分类只在地面清洁通过后显示)
26. 验证证据要求
27. 点击 "完成并下一个"
28. 自动跳转到下一个目标
29. 打完所有目标

### Phase 5: 检查员 — 提交
30. 点击 "提交任务"
31. 确认提交

### Phase 6: 验证结果
32. 返回项目管理查看完成状态
33. 查看得分和等级
