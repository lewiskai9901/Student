## 1. 执行摘要

### 一句话
**哨声 AI** 是一款面向中小学体育老师的 AI 智能助教 Pad 应用——一台设备、一节课、一份全班报告。

### 痛点
全国 55 万体育老师每天在重复做三件非教学工作:**点名、计数、写报告**,占用 40 分钟课时中的 **16 分钟(40%)**。现有解决方案(体测机、穿戴设备、单项 App)频次低、成本高、覆盖窄。

### 产品
一台 Pad 架在操场,端侧 AI 完成三个核心闭环:
1. **AI 点名考勤**(1 分钟全班完成)
2. **群体动作识别 + 个体计数**(业界首创,非单人计数)
3. **一键课堂报告 + 学生长期画像**(LLM 自动撰写,家长小程序推送)

### 市场
- TAM:K12 体育科技 **1200 亿**(艾瑞 2025),年复合增速 28%
- SAM:55 万中小学体育老师 + 80 万体培教练 + 30 万幼师 = **165 万教师**
- SOM:Y3 目标老师渗透 36%(20 万)、学校渗透 0.9%(2000 所)

### 商业模式
三级火箭漏斗:
- **¥299/年** 教师个人订阅(绕开招标,直达用户)
- **¥3-8 万/年** 学校版(数据资产化,校长驾驶舱)
- **¥30-100 万/年** 区域版(教育局级体质预警平台)

### 核心创新
- 🥇 **第一家"教师订阅"切入校园 AI** — 绕开招标泥潭
- 🥇 **第一家做"群体动作识别"** — 业界都在做单人计数
- 🥇 **第一家建设体育课堂多模态数据集** — 时间壁垒,资产化护城河

### 增长指标(三年)

## 3. 用户与痛点

### 核心用户画像:王老师(中学体育老师)

- 38 岁,男,省会二线城市公立中学,教 8 个班,每周 24 节课
- 月薪 8000,年绩效考核看学生体测合格率与家长满意度
- 每天花 1-2 小时写教案、填报表、回复家长
- 会用微信和钉钉,不怕新工具,但讨厌"复杂/难配/要学 3 小时"的产品

### 老师一节课的真实时间分布(40 分钟)

| 环节 | 时长 | 是否非教学 |
|---|---|---|
| 课前点名 | 5 分钟 | ✅ 非教学 |
| 热身讲解 | 5 分钟 | 教学 |
| 分组练习 + 计数 | 15 分钟 | 其中 5 分钟在数人头 |
| 自由活动观察 | 10 分钟 | 其中 3 分钟在记录 |
| 课后总结 | 5 分钟 | ✅ 非教学 |
| **课后填报表(非课内)** | 30-60 分钟/天 | ✅ 非教学 |

**结论**:40 分钟课 + 30-60 分钟课后,**非教学劳动占 40%-50%**。这是我们要还给老师的时间。

### 访谈验证(2026 年 3 月)

- 访谈样本:40 位中小学体育老师(北京/上海/杭州/武汉/成都)
- **25 位 (62.5%)** 明确愿意为解决"点名/计数/报告"付费
- 平均可接受月订阅价格:**¥38/月**(我们定价 ¥29 在其心理价位内)
- 最期待功能排序:课后报告 (82%) > 自动计数 (75%) > 点名 (68%) > 家长推送 (60%)

### 间接受益人

| 角色 | 价值 |
|---|---|
| **学生** | 获得客观公平的过程性评价,而非老师印象打分 |
| **家长** | 通过小程序看到孩子每节体育课的运动数据、趋势、健康建议 |
| **校长** | 拥有全校体质数据驾驶舱,绩效考核、家长沟通、上级汇报有据 |
| **教育局** | 区域学生体质大数据,政策制定、预警、评估的基础设施 |

## 4. 产品与解决方案

### 产品形态

- **端侧**:iPad / 安卓 Pad App(离线优先,操场无 WiFi 也能用)
- **云端**:多租户 SaaS 后台(教师 / 学校 / 区域三级权限)
- **家长端**:微信小程序(学生画像、周报推送)

### 核心功能闭环

#### 功能一:AI 智能点名考勤

- 上课前将 Pad 架三脚架,学生依次通过或集合,自动人脸识别完成点名
- 缺勤/迟到自动标记并推送班主任
- 技术:InsightFace 端侧人脸识别,30ms/人,精度 99%+

#### 功能二:AI 群体动作识别 + 个体计数

三大场景:

| 场景 | 内容 | 示例输出 |
|---|---|---|
| 热身/做操 | 全班队形整齐度、动作同步率 | "队形整齐度 85 分,第 3 排 2 人动作不到位" |
| 分组练习 | 按学号计数(跳绳/俯卧撑/折返跑) | "张三 1 分钟跳绳 112 次,李四 98 次" |
| 自由活动 | 骨骼点位移量 + 运动强度估算 | "王五运动参与度 72%,低于班均" |

- 技术:YOLOv8 检测 + ByteTrack 追踪 + RTMPose 骨骼 + **自研 ST-GCN 动作分类头**(护城河)

#### 功能三:一键课堂报告 + 学生长期画像

- 下课即生成班级报告(强度、达标率、亮点、待关注)
- 周/月自动汇总,学生运动趋势曲线,对比班级均值
- 家长小程序推送(病毒传播点)

### 交互原型关键决策

- **零学习成本**:老师只需点"开始上课"一个按钮,下课自动收工
- **人机协同兜底**:AI 识别有疑义时标记,老师一键确认或纠正,确认数据自动回流训练集
- **隐私优先**:学生视频/人脸模板**仅存本地 Pad 加密存储**,云端仅接收结构化脱敏数据

### 明确不做(YAGNI)

- ❌ 技术动作纠错(投篮/击球姿势)——精度难达 90%+,容易翻车
- ❌ 心率监测——没穿戴设备做不到精准,用运动量估算代替
- ❌ 多摄像头/边缘盒子——留到 B 轮做校园版
- ❌ 教育局级大数据平台——没学校规模前没意义

## 5. 核心技术与护城河

### 技术架构(三层)

```
端侧 App(iPad/安卓 Pad)——离线优先
   ↓ 仅上传结构化数据(非视频)
云端 SaaS 数据中台
   ↓
AI 训练平台(数据飞轮)
```

### 技术选型:能用开源绝不自研

| 模块 | 方案 | 自研/开源 |
|---|---|---|
| 人脸识别 | InsightFace | 开源 |
| 人体检测 | YOLOv8-nano | 开源 |
| 多人追踪 | ByteTrack | 开源 |
| 骨骼点 | RTMPose | 开源 |
| **动作分类** | **ST-GCN + 自研分类头** | **自研** |
| **群体动作识别** | **自研** | **自研** |
| 端侧推理 | MNN / NCNN 量化 | 开源 |
| 报告生成 | 通义/豆包 API | 调用 |

### 三大护城河

#### 1. 场景数据集(时间壁垒)
- Phase 0:3 所样板校自采,3 万条动作样本
- Phase 1:老师反馈闭环,用越多越准
- Phase 2:**全国首个体育课堂多模态数据集** — 大厂拿钱买不到

#### 2. 群体动作识别算法(技术壁垒)
- 业界公开方案都是单人计数(跳绳 App、引体向上 App)
- **群体队形/整齐度/互动**是我们独有的研究方向,2-3 年技术领先

#### 3. 教师社区 + 教研内容生态(品牌壁垒)
- 大厂做通用视觉中台,我们做"懂体育老师的场景 know-how"
- 教师社区每日产出 UGC 教研内容,长期积累形成网络效应

### 关键指标

| 指标 | 目标 |
|---|---|
| 端侧推理帧率 | 30 fps(Pad 端) |
| 动作识别准确率 | 跳绳 95%+ / 广播操整齐度 90%+ |
| 模型大小 | < 50 MB(iPad 2021 可跑) |
| 单课时功耗 | < 15% 电量 / 40 分钟 |
