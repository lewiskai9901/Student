# 「我的班级」班主任工作台设计方案

> 创建日期: 2026-01-18
> 状态: 待实施

## 一、功能概述

### 1.1 背景

当前系统是面向管理员设计的后台管理系统，班主任/辅导员等用户使用时需要在各个模块之间跳转，缺少聚合视图。本设计旨在为管理班级的用户提供专属工作台。

### 1.2 目标用户

- 班主任
- 副班主任
- 辅导员
- 任课教师
- 年级主任
- **任何被分配了班级的用户**

### 1.3 核心需求

- 日常管理：快速了解班级状态、处理待办事项
- 数据分析：分析班级整体表现、发现问题学生
- 多班级支持：一个用户可管理多个班级

## 二、整体架构

```
我的班级
├── 班级概览页（入口）
│   └── 多班级卡片展示 / 单班级直接进入
├── 班级详情页
│   ├── 📊 概览 Dashboard
│   ├── 👥 学生管理
│   ├── 🏠 宿舍分布
│   └── 📈 数据分析
```

### 2.1 访问逻辑

```
用户登录 → 检查是否有分配的班级 → 有则显示「我的班级」菜单
```

### 2.2 入口设计

**方案：智能首页 + 独立菜单**

1. **首页智能切换**：班主任登录后首页显示本班概览，管理员看全校数据
2. **独立菜单入口**：侧边栏新增「我的班级」一级菜单（仅有班级的用户可见）

```
📊 首页（智能切换内容）
🎓 我的班级 ← 新增（有班级的用户可见）
📁 组织管理
✅ 量化检查
...
```

## 四、技术设计

### 4.1 前端路由

```typescript
// router/index.ts
{
  path: '/my-class',
  name: 'MyClass',
  meta: {
    title: '我的班级',
    requiresClass: true  // 需要有分配班级
  },
  children: [
    { path: '', name: 'MyClassList', component: MyClassListView },
    { path: ':classId', name: 'MyClassDetail', component: MyClassDetailView }
  ]
}
```

### 4.2 后端接口

```
GET  /api/v2/my-class/classes           # 获取用户管理的班级列表
GET  /api/v2/my-class/:classId/overview # 班级概览数据
GET  /api/v2/my-class/:classId/students # 班级学生列表
GET  /api/v2/my-class/:classId/dormitory-distribution  # 宿舍分布
GET  /api/v2/my-class/:classId/analytics # 数据分析
```

### 4.3 数据权限

- 复用现有的 `TeacherAssignment` 关联
- 用户只能访问自己被分配的班级
- 后端验证：检查当前用户是否在该班级的 teacherAssignments 中

### 4.4 菜单权限

```typescript
// 菜单生成逻辑
if (user.assignedClasses && user.assignedClasses.length > 0) {
  // 显示「我的班级」菜单
}
```

## 五、交互规范

| 交互场景 | 设计方案 |
|---------|---------|
| 班级卡片悬停 | 上浮4px + 阴影加深 + 显示"进入"按钮 |
| 班级切换 | 下拉选择器，带搜索，显示班级人数 |
| 数据加载 | 骨架屏占位，避免布局跳动 |
| 空状态 | 友好插图 + 引导文案 |
| 宿舍点击 | 弹出抽屉显示该宿舍学生列表 |
