## 1. 产品定位与核心价值

### 产品名
**哨声 AI** — 一声哨响,全班数据自动记录

### 一句话定位
面向中小学体育老师的 AI 智能助教 Pad 应用,一台设备完成一节课的全部记录、评价与报告工作。

### 核心用户
- **主购买决策人**:中小学体育老师(全国 55 万人)
- **间接受益人**:学生(公平评价)、家长(孩子运动画像)、校长/教育局(体质数据资产)

### 价值主张(给老师讲的话)
> 以前一节体育课 40 分钟,你要喊口令、点名、数数、记分、课后还要填表。
> 现在把 Pad 架在旁边,下课自动生成全班考勤、运动量、动作达标率、一周趋势报告——你只管教。

### 三句话差异化
1. **不是体测机**:体测机一年用一次,我们是每节课都用的教学工具
2. **不是穿戴设备**:零学生端硬件,零采购阻力,老师自己就能决定用不用
3. **不是单项计数 App**:市面上有跳绳/引体向上的单点 App,我们是一整节课的 AI 助教

## 2. MVP 核心功能设计

遵循 YAGNI 原则,MVP 只做 3 个功能闭环。

### 功能一:AI 智能点名考勤(入口钩子)
- 上课前把 Pad 架好,学生依次从镜头前走过或集合站队,Pad 自动人脸识别完成点名
- 1 分钟内出考勤表,缺勤/迟到自动标记、一键推送班主任
- 技术:端侧人脸识别(InsightFace 开源)

### 功能二:AI 群体动作识别 + 个体计数(核心差异化)
- **场景 A - 热身/做操**:识别全班动作整齐度,给出"队形整齐度 85 分""第 3 排有 2 人动作不到位"
- **场景 B - 分组练习**:老师用 Pad 对准某组,自动数跳绳次数、俯卧撑数、折返跑圈数,按学号归档
- **场景 C - 自由活动/游戏**:识别每个学生的活动量(骨骼点位移量 + 心率估计),输出"运动参与度"
- 技术:YOLOv8 + ByteTrack + RTMPose + 自研动作分类头

### 功能三:课后一键报告 + 学生长期画像(留存与付费驱动)
- 课堂结束自动生成班级报告(整体运动强度、达标率、亮点学生、待关注学生)
- 周/月报告自动生成,每个学生运动趋势曲线,自动对比班级均值
- 家长端小程序(病毒传播点:一个老师用 → 全班家长看到 → 其他班老师被问"为什么我们班没有")

### MVP 明确不做(YAGNI)
- ❌ 技术动作纠错("你的投篮姿势不标准")——太难做准,容易翻车
- ❌ 心率监测(没穿戴设备做不到精准)——用运动量估算代替
- ❌ 自动编排训练计划——交给老师专业判断
- ❌ 多摄像头/边缘盒子——留到 B 轮做校园版
- ❌ 对接教育局大数据平台——没学校规模之前没意义

## 3. 商业模式与三年路径

### 商业模式画布

| 维度 | 设计 |
|---|---|
| 客户细分 | 核心:55 万中小学体育老师;拓展:80 万体培教练 + 30 万幼师 |
| 价值主张 | 教师端:省时 + 绩效有据;学校端:体质数据资产化;家长端:孩子运动可视化 |
| 渠道 | 一阶段:抖音/小红书教师 KOL + 教研员社群;二阶段:地推进教研室;三阶段:省市教育局 |
| 收入模式 | 三层订阅 + 硬件租赁 + 数据服务 |
| 关键资源 | 自研动作识别模型 + 全国首个"体育课多模态数据集" |
| 关键活动 | 算法迭代 / 教师社区运营 / 教研内容生产 |
| 成本结构 | 研发 60% / 获客 25% / 云+算力 10% / 其他 5% |

### 三档订阅漏斗

| 版本 | 价格 | 目标用户 | 功能 |
|---|---|---|---|
| 免费版 | ¥0 | 体验用户 | 点名 + 3 次/月 AI 识别 |
| 老师版 | ¥29/月 或 ¥299/年 | 个体老师自费 | 无限课次 + 全部 AI 识别 + 家长小程序 |
| 学校版 | ¥3-8 万/年 | 学校统一采购 | 多老师协作 + 全校体质档案 + 校长驾驶舱 |
| 区域版 | ¥30-100 万/年 | 区/市教育局 | 跨校分析 + 体质预警 + 省级体测对接 |

### 三年财务预估(保守版)

| 年度 | 付费老师 | 付费学校 | 区域客户 | ARR(万元) | 关键动作 |
|---|---|---|---|---|---|
| Y1 (MVP 验证) | 5,000 | 20 | 0 | ~280 | 单城市打穿 50 所学校 |
| Y2 (规模化) | 50,000 | 300 | 3 | ~3,000 | 5 省教研合作 + A 轮融资 |
| Y3 (平台化) | 200,000 | 2,000 | 20 | ~25,000 | 横向拓展体培 + 幼儿园 |

渗透率锚点:全国 22 万所中小学,Y3 学校渗透 0.9%,老师渗透 36%——对标钉钉班级群早期渗透速度可行。

## 4. 技术实现路径与团队配置

### 技术架构(三层)

```
┌─────────────────────────────────────────────┐
│ 端侧 App (iPad/安卓 Pad) — 离线优先           │
│  ├─ 视频采集 & 预处理                          │
│  ├─ 端侧推理引擎 (CoreML / NCNN / MNN)        │
│  ├─ 本地加密存储 (学生人脸/视频不上云)          │
│  └─ 老师交互 UI (一键开课/标记/导出)           │
└───────────────┬─────────────────────────────┘
                │ 仅上传结构化数据(非视频)
┌───────────────▼─────────────────────────────┐
│ 云端 SaaS — 数据中台                         │
│  ├─ 报告生成引擎 (LLM 自动撰写)               │
│  ├─ 学生画像/趋势分析                         │
│  ├─ 家长小程序 / 校长驾驶舱                   │
│  └─ 多租户权限 (教师/学校/区域)               │
└───────────────┬─────────────────────────────┘
                │
┌───────────────▼─────────────────────────────┐
│ AI 训练平台 — 数据飞轮                       │
│  ├─ 老师标注反馈闭环 (错判一键纠正)           │
│  ├─ 体育动作数据集 (护城河)                   │
│  └─ 模型 OTA 下发                            │
└─────────────────────────────────────────────┘
```

### 技术选型(能用开源绝不自研)

| 模块 | 方案 | 自研/开源 | 备注 |
|---|---|---|---|
| 人脸识别考勤 | InsightFace / ArcFace | 开源 | 端侧 30ms,精度 99%+ |
| 人体检测 | YOLOv8-nano | 开源 | Pad 端实时 30fps |
| 多人追踪 | ByteTrack | 开源 | 防止重复计数 |
| 骨骼点 | RTMPose / MediaPipe | 开源 | 17/33 关键点 |
| 动作分类 | ST-GCN + 自研分类头 | **自研** | 核心护城河 |
| 群体动作识别 | 自研 | **自研** | 差异化亮点 |
| 报告文本生成 | 通义 / 豆包 API | 调用 | 省成本 |
| 端侧推理 | MNN / NCNN + 模型量化 | 开源 | Pad 端跑得动关键 |

### 数据飞轮(大赛讲故事关键)
1. **Phase 0 (0-3月)**:自采——3 所合作学校,录 100 节课视频,人工标注 3 万条动作样本
2. **Phase 1 (4-12月)**:老师标注反馈——AI 识错一键纠正,自动回流训练集
3. **Phase 2 (Y2+)**:形成全国首个体育课堂多模态数据集,可融资的资产

### 技术风险 & 应对

| 风险 | 可能性 | 应对 |
|---|---|---|
| 操场光照/逆光精度下降 | 高 | 数据增强 + 老师纠错 + 架设指南 |
| 多学生同服饰难区分 | 中 | ReID 模型 + 胸前号码牌 |
| 学生家长隐私顾虑 | **高** | **端侧处理、视频不出 Pad、只传脱敏数据** |
| Pad 算力不够 | 中 | 模型蒸馏到 <50M,最低 iPad 2021 款可跑 |
| 大厂下场 | 中 | 教师社区 + 教研内容建立品牌壁垒 |

### MVP 工期与团队配置

**90 天 MVP 目标**:1 所合作学校、10 个老师、2 种运动类型(跳绳 + 广播体操)跑通全链路

| 角色 | 人数 | 职责 |
|---|---|---|
| CEO/产品 | 1 | 学校 BD、老师访谈、融资、产品决策 |
| CTO/算法 | 1 | 动作识别模型、端侧部署 |
| 客户端工程师 | 1 | Pad App、端侧推理集成 |
| 后端/全栈 | 1 | SaaS 后台、家长小程序 |
| 算法工程师(初级) | 1 | 数据标注、模型训练 |
| 体育顾问(兼职) | 1 | 体育教研员/名师,内容+渠道 |

**启动资金**:种子轮 300-500 万,支撑 12 个月到 A 轮

### 60 秒电梯稿

### 评委必问 8 问

| # | 问题 | 标准回答核心 |
|---|---|---|
| 1 | 为什么是你们做,不是商汤/讯飞? | 大厂做通用平台,我们做懂体育老师的场景 know-how;大厂没有老师标注飞轮的耐心 |
| 2 | 学生家长会不会有隐私顾虑? | 视频端侧零上云 + 人脸模板加密本地 Pad + 家长授权才生成画像——是亮点不是包袱 |
| 3 | 老师真的会付费吗? | 访谈 40 位,25 位愿付;对标"班级小管家"免费工具做到年 GMV 数亿 |
| 4 | 政策合规? | 不涉及学科培训,是政策鼓励方向(2024《学校体育工作评估办法》要求过程性数据) |
| 5 | 为什么不做穿戴? | 硬件拖死早期团队(库存/售后);软件跑通 1 万老师再做硬件,单客 LTV 更高 |
| 6 | 技术壁垒到底是什么? | 不是单点模型,而是场景数据集 + 反馈闭环 + 教研生态三位一体——时间壁垒 |
| 7 | 省把它做成免费公益怎么办? | 正是我们要的——教育局买断我们区域版给老师免费用,是 2B2G 最优出口 |
| 8 | 天花板在哪? | 165 万教师 TAM + 幼儿体能/运动康复/赛事分析横向拓展,百亿级天花板 |

### 反脆弱设计

| 场景 | 预案 |
|---|---|
| 大厂免费抄袭 | 靠教师社区 + 教研内容 + 名师代言品牌壁垒 |
| 算法暂不到位 | "人机协同"话术:AI 初判 + 老师确认,精度 70% 也能用,每次确认训练模型 |
| 政策收紧校园 AI | 快速切课外体培 + 幼儿园(同算法不同渠道) |
| 学校 Pad 不够 | Pad + App 整包硬件租赁(¥199/月含设备),学校零采购压力 |
| 融资不顺 | 前 500 个教师订阅现金流可养 3 人小团队,Day 1 非烧钱模式 |

### 三个创新点(PPT 封面)
- 🎯 **第一个"教师订阅"切入校园 AI 的公司** — 绕开招标泥潭
- 🎯 **第一个做"群体动作识别"的体育 AI** — 业界都在做单人计数
- 🎯 **第一个建设体育课堂多模态数据集的团队** — 时间壁垒 + 资产化

### 大赛加分话术
- **金句一**:"体测机是体检报告,我们是每日手环——高频才是教育的真刚需"
- **金句二**:"一节 40 分钟体育课,老师花 16 分钟做非教学工作,我们还他们这 16 分钟"
- **金句三**:"让 55 万体育老师,重新爱上上课"

| 9 | **深兰科技 AI 体育** | AI 裁判/赛事分析 | 技术秀肌肉 | 落地难、故事大于产品 | 方向不同 |

### 关键判断
- **没有领头羊**:赛道分散,第一名市占率不足 5%,窗口期还在
- **大厂基因不合**:商汤/讯飞不愿意做"50 元订阅"的苦活累活
- **教师订阅 + 群体识别**两个组合拳,10 家里**没人同时做**
- **可能威胁**:猿力 Keep 若下定决心做校园,流量优势大——我们必须用**教师社区 + 教研内容**反向拉开身位

## 9. 方向横向拓展

同一个"AI 摄像头 + 骨骼识别 + 报告引擎"内核,可平移到:

1. **幼儿园体能课** — 监测 3-6 岁大运动发育里程碑(跳、跑、平衡),对接儿保系统,家长付费意愿极强(幼儿园家长客单价是小学 3 倍)
2. **课外体培机构**(篮球/足球/羽毛球) — 把动作纠错做进去,教练用来做教学成果可视化,转化为续费工具,机构按课时分成
3. **课间操 / 大课间** — 替换目前"眼保健操摄像头"赛道(该赛道已有公司年收入过亿但产品很糙)
4. **家庭亲子运动** — 小程序/TV 端,周末作业打卡,反向从家长端拉新学校
5. **青少年运动康复** — 与医院儿科/骨科合作,脊柱侧弯/扁平足/步态异常筛查(客单价最高,但医疗准入周期长)
6. **赛事与校运会** — 学校运动会 AI 裁判 / 即时回放 / 自动颁奖视频 —— **超强传播点**,一次运动会一条爆款短视频

### 拓展策略
- **主线死磕**:方向 1-3(学校课堂)
- **获客钩子**:方向 6(校运会) — 免费送学校办一次运动会,换取常态化体育课使用权
- **Y3 横向扩张**:方向 5(运动康复) — 毛利最高、对 B 单价最大
- **不碰**:方向 4(家庭) — 获客成本太高,除非 Keep 收购我们顺带整合
