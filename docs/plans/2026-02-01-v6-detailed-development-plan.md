## 一、命名规范统一方案

### 1.1 统一标准

| 层级 | 旧命名 | 新命名 | 说明 |
|------|-------|-------|------|
| **概念** | 部门/Department | 组织/Organization | 更通用，适配多行业 |
| **数据库表** | departments | org_units | 已存在，需迁移数据 |
| **表字段** | dept_code, dept_name | unit_code, unit_name | 统一前缀 |
| **后端实体** | OrgUnitPO | OrgUnitPO | 保持不变 |
| **前端** | DepartmentsView | OrganizationView | 重命名 |
| **API路径** | /departments | /org-units | 已完成 |
| **权限** | system:department:* | system:org:* | 统一 |

### 1.2 命名改造清单

```
需要改造的文件/模块：

后端：
├── OrgUnitPO.java              → 改表映射和字段名
├── DepartmentRankingController → 改名为 OrgRankingController
├── DepartmentRankingService    → 改名为 OrgRankingService
├── OrgUnitType枚举             → 重新定义（配合配置化改造）
└── 权限定义                     → 统一为 system:org:*

前端：
├── DepartmentsView.vue         → OrganizationView.vue
├── departmentRanking.ts        → orgRanking.ts
├── 路由配置                     → 更新菜单名称
└── 权限配置                     → 更新权限代码

数据库：
├── departments表               → 数据迁移到org_units
├── user_departments表          → 迁移到user_org_relations
└── 权限种子数据                 → 更新权限代码

小程序：
├── department.ts               → organization.ts
└── 相关类型定义                 → 统一命名
```

#### 迁移脚本示例

```sql
-- V8.0.0__naming_unification.sql

-- 1. 确保 org_units 表存在并包含所有字段
-- 2. 迁移 departments 数据到 org_units（如果尚未迁移）
-- 3. 重命名字段
-- 4. 更新权限数据

-- 权限代码更新
UPDATE permissions SET permission_code = REPLACE(permission_code, 'department', 'org')
WHERE permission_code LIKE '%department%';

-- 角色权限关联更新
UPDATE role_permissions rp
INNER JOIN permissions p ON rp.permission_id = p.id
SET rp.permission_id = rp.permission_id; -- 触发更新

-- 菜单名称更新
UPDATE menus SET menu_name = '组织管理' WHERE menu_name = '部门管理';
UPDATE menus SET menu_name = '组织架构' WHERE menu_name = '部门架构';
```
