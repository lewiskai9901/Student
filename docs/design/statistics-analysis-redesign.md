# 统计分析系统重构设计方案

## 一、问题分析

### 1.1 当前问题

| 问题 | 描述 | 影响 |
|------|------|------|
| **检查覆盖不一致** | 某班级在某次检查中未被检查 | 统计总扣分时产生不公平比较 |
| **模板不一致** | 不同检查使用不同模板 | 类别统计无法跨检查聚合 |
| **类别不一致** | 检查类别因模板而异 | 无法进行统一的类别分析 |
| **统计不灵活** | 固定的统计维度 | 无法满足多样化分析需求 |
| **范围确定困难** | 按时间 vs 按记录 | 两种方式各有优劣 |

### 1.2 核心设计原则

参考 [Dashboard Design Patterns](https://dashboarddesignpatterns.github.io/patterns.html) 和 [Databox Report Builder](https://databox.com/report-software/saas) 的设计理念：

1. **数据标准化** - 使用相对指标而非绝对指标
2. **配置驱动** - 统计逻辑由配置定义，而非硬编码
3. **渐进披露** - 简单场景简单操作，复杂场景支持高级配置
4. **实时与快照** - 同时支持动态更新和静态报告

## 二、解决方案：检查覆盖不一致问题

### 2.1 核心指标设计

```
┌─────────────────────────────────────────────────────────────────┐
│                      统计指标体系                                │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  绝对指标（受覆盖率影响）          相对指标（消除覆盖率影响）      │
│  ─────────────────────          ─────────────────────────      │
│  • 总扣分                        • 平均扣分 = 总扣分/检查次数     │
│  • 总检查次数                    • 扣分率 = 总扣分/最大可能扣分   │
│  • 扣分项总数                    • 覆盖率 = 实际检查/应检查次数   │
│                                  • 标准化得分 = 扣分×覆盖率权重   │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 2.2 覆盖率加权算法

```
标准化扣分 = 平均扣分 × (实际检查次数 / 最大检查次数)

示例：
- 班级A：总扣分 30，检查 10 次，平均扣分 3.0
- 班级B：总扣分 24，检查 8 次，平均扣分 3.0
- 最大检查次数：10 次

班级A 标准化扣分 = 3.0 × (10/10) = 3.0
班级B 标准化扣分 = 3.0 × (8/10) = 2.4  ← 因少检查2次，降低权重
```

### 2.3 处理策略选项

用户可选择以下策略之一：

| 策略 | 适用场景 | 计算方式 |
|------|---------|---------|
| **平均扣分** | 公平比较 | 总扣分 ÷ 检查次数 |
| **加权平均** | 考虑覆盖率 | 平均扣分 × 覆盖率系数 |
| **仅全覆盖** | 严格统计 | 只统计全部被检查的班级 |
| **缺检惩罚** | 强调出勤 | 缺检按最高扣分计算 |
| **缺检豁免** | 宽松统计 | 缺检不计入统计 |

## 三、解决方案：范围确定方式

### 3.1 双模式设计

```
┌─────────────────────────────────────────────────────────────────┐
│                    分析范围配置                                   │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  ┌─────────────────┐        ┌─────────────────┐                │
│  │   时间范围模式   │   OR   │   记录选择模式   │                │
│  └────────┬────────┘        └────────┬────────┘                │
│           │                          │                          │
│     ┌─────┴─────┐              ┌─────┴─────┐                   │
│     │           │              │           │                   │
│  固定范围    动态范围      手动选择    条件筛选                  │
│  (起止日期)  (起始+无终止)  (勾选记录)  (自动匹配)               │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 3.2 时间范围模式

```typescript
interface TimeRangeConfig {
  type: 'fixed' | 'dynamic';
  startDate: Date;           // 起始日期
  endDate?: Date;            // 终止日期（动态模式为空）
  includeStartDay: boolean;  // 是否包含起始日
  includeEndDay: boolean;    // 是否包含终止日
  autoRefresh: boolean;      // 是否自动刷新
  refreshInterval?: number;  // 刷新间隔（分钟）
}
```

### 3.3 记录选择模式

```typescript
interface RecordSelectConfig {
  mode: 'manual' | 'condition';

  // 手动模式：直接选择记录ID
  selectedRecordIds?: number[];

  // 条件模式：自动匹配
  conditions?: {
    dateRange?: { start: Date; end: Date };
    checkTypes?: string[];     // 检查类型
    templateIds?: number[];    // 模板ID
    statusList?: number[];     // 状态（已发布、已审核等）
  };
}
```

### 3.4 推荐方案

**建议采用「时间范围 + 条件筛选」混合模式**：

1. 主要使用时间范围确定大致范围
2. 辅以条件筛选排除/包含特定记录
3. 提供预览功能，显示将被纳入统计的记录列表
4. 支持手动微调（排除异常记录）

### 4.2 数据模型设计

#### 4.2.1 分析配置主表

```sql
CREATE TABLE analysis_configs (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    plan_id BIGINT NOT NULL COMMENT '所属检查计划ID',
    config_name VARCHAR(100) NOT NULL COMMENT '配置名称',
    config_desc VARCHAR(500) COMMENT '配置描述',

    -- 范围配置
    scope_type ENUM('time', 'record', 'mixed') DEFAULT 'time' COMMENT '范围类型',
    scope_config JSON COMMENT '范围配置JSON',

    -- 目标配置
    target_config JSON COMMENT '目标配置JSON',

    -- 更新模式
    update_mode ENUM('static', 'dynamic') DEFAULT 'static' COMMENT '更新模式',
    auto_refresh BOOLEAN DEFAULT FALSE COMMENT '是否自动刷新',
    refresh_interval INT COMMENT '刷新间隔(分钟)',
    last_refresh_time DATETIME COMMENT '上次刷新时间',

    -- 缺检处理策略
    missing_strategy ENUM('avg', 'weighted', 'full_only', 'penalty', 'exempt')
        DEFAULT 'avg' COMMENT '缺检处理策略',

    -- 通用字段
    is_enabled BOOLEAN DEFAULT TRUE,
    is_public BOOLEAN DEFAULT FALSE COMMENT '是否公开(其他用户可见)',
    sort_order INT DEFAULT 0,
    creator_id BIGINT,
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    updated_at DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    deleted BOOLEAN DEFAULT FALSE,

    INDEX idx_plan_id (plan_id),
    INDEX idx_creator (creator_id)
) COMMENT '统计分析配置表';
```

#### 4.2.2 统计指标配置表

```sql
CREATE TABLE analysis_metrics (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    config_id BIGINT NOT NULL COMMENT '分析配置ID',
    metric_name VARCHAR(100) NOT NULL COMMENT '指标名称',
    metric_desc VARCHAR(500) COMMENT '指标描述',

    -- 指标类型
    metric_type ENUM(
        'total_score',        -- 总扣分
        'avg_score',          -- 平均扣分
        'check_count',        -- 检查次数
        'coverage_rate',      -- 覆盖率
        'item_count',         -- 扣分项次数
        'person_count',       -- 涉及人次
        'weighted_score',     -- 加权扣分
        'ranking',            -- 排名
        'trend',              -- 趋势
        'custom'              -- 自定义公式
    ) NOT NULL COMMENT '指标类型',

    -- 数据来源
    source_type ENUM('all', 'category', 'item') DEFAULT 'all' COMMENT '数据来源类型',
    source_ids JSON COMMENT '来源ID列表(类别ID或扣分项ID)',

    -- 聚合方式
    aggregation ENUM('sum', 'avg', 'max', 'min', 'count') DEFAULT 'sum',

    -- 自定义公式
    custom_formula VARCHAR(500) COMMENT '自定义计算公式',

    -- 显示配置
    display_format VARCHAR(50) DEFAULT '{value}' COMMENT '显示格式',
    decimal_places INT DEFAULT 2 COMMENT '小数位数',
    unit VARCHAR(20) COMMENT '单位(分、次、%等)',

    -- 图表配置
    chart_type ENUM('number', 'bar', 'line', 'pie', 'table', 'rank') DEFAULT 'number',
    chart_config JSON COMMENT '图表配置JSON',

    -- 排序和高亮
    sort_field VARCHAR(50) COMMENT '排序字段',
    sort_order ENUM('asc', 'desc') DEFAULT 'asc',
    highlight_rule JSON COMMENT '高亮规则(阈值着色)',

    -- 通用字段
    is_enabled BOOLEAN DEFAULT TRUE,
    display_order INT DEFAULT 0,
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,

    INDEX idx_config_id (config_id),
    FOREIGN KEY (config_id) REFERENCES analysis_configs(id) ON DELETE CASCADE
) COMMENT '统计指标配置表';
```

#### 4.2.3 分析目标配置

```sql
-- 目标配置存储在 analysis_configs.target_config JSON 字段中

{
```

#### 4.2.4 分析结果快照表

```sql
CREATE TABLE analysis_snapshots (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    config_id BIGINT NOT NULL COMMENT '分析配置ID',
    snapshot_name VARCHAR(100) COMMENT '快照名称',

    -- 快照数据
    snapshot_data JSON NOT NULL COMMENT '快照数据',

    -- 统计信息
    record_count INT COMMENT '纳入统计的记录数',
    class_count INT COMMENT '涉及班级数',
    date_range_start DATE COMMENT '数据起始日期',
    date_range_end DATE COMMENT '数据截止日期',

    -- 元数据
    generated_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    generated_by BIGINT,
    is_manual BOOLEAN DEFAULT FALSE COMMENT '是否手动生成',

    INDEX idx_config_id (config_id),
    FOREIGN KEY (config_id) REFERENCES analysis_configs(id) ON DELETE CASCADE
) COMMENT '分析结果快照表';
```

### 4.3 类别映射设计

为解决不同模板类别不一致的问题，引入统一类别映射：

```sql
CREATE TABLE category_mappings (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    plan_id BIGINT NOT NULL COMMENT '检查计划ID',
    unified_category_code VARCHAR(50) NOT NULL COMMENT '统一类别编码',
    unified_category_name VARCHAR(100) NOT NULL COMMENT '统一类别名称',

    -- 映射的原始类别
    source_category_ids JSON NOT NULL COMMENT '源类别ID列表',

    -- 显示配置
    color VARCHAR(20) COMMENT '显示颜色',
    icon VARCHAR(50) COMMENT '图标',
    sort_order INT DEFAULT 0,

    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,

    INDEX idx_plan_id (plan_id),
    UNIQUE KEY uk_plan_category (plan_id, unified_category_code)
) COMMENT '类别映射表';
```

```java
@Service
public class MetricCalculationEngine {

    public MetricResult calculate(AnalysisMetric metric, NormalizedData data) {
        // 1. 筛选数据来源
        List<DataPoint> sourceData = filterBySource(data, metric.getSourceConfig());

        // 2. 聚合计算
        Map<Long, BigDecimal> aggregatedData = aggregate(sourceData, metric.getAggregation());

        // 3. 排序
        List<RankedItem> ranked = rank(aggregatedData, metric.getSortConfig());

        // 4. 应用高亮规则
        applyHighlight(ranked, metric.getHighlightRule());

        // 5. 构建图表数据
        ChartData chartData = buildChartData(ranked, metric.getChartType());

        return new MetricResult(metric, ranked, chartData);
    }
}
```
