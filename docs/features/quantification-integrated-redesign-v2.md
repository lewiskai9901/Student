## 二、重构目标

### 2.1 核心原则

```

```

| 策略 | 说明 |

| 保留核心实体 | CheckExportTemplate, CheckPlanRatingRule 等实体设计良好 |

## 六、关键问题与回答

### Q1: 轮次是在模板中定义还是每次检查动态定义？

**回答：每次检查动态定义**

```
CheckTemplate.totalRounds / roundNames  →  定义默认轮次
DailyCheck.totalRounds / roundNames     →  每次检查可覆盖

当前已实现：DailyCheck 有自己的 totalRounds 和 roundNames 字段
```

### Q2: 导出时如何选择轮次？

**回答：动态获取当前检查的轮次列表**

```javascript
// 前端获取检查的轮次
const rounds = JSON.parse(dailyCheck.roundNames) // ["早检", "午检", "晚检"]

// 用户勾选要导出的轮次
const selectedRounds = [1, 2] // 第1轮和第2轮
```

### Q3: 多检查汇总时，如何确保检查结构一致？

**回答：它们都属于同一个检查计划，使用相同的模板**

```
CheckPlan.templateId → 所有 DailyCheck 使用相同的检查模板
因此扣分项、类别结构都是一致的，只是轮次数量可能不同
```

### Q4: 评级是自动触发还是手动触发？

**建议：手动触发 + 可选自动**

```
CheckPlanRatingRule.autoCalculate = 0  →  手动触发
CheckPlanRatingRule.autoCalculate = 1  →  检查发布后自动评级
```

### Q5: 评级结果可以修改吗？

**当前设计：可以**

```
CheckPlanRatingResult.version        →  版本号
CheckPlanRatingResult.originalLevelId →  原始等级（记录修改前）
```

## 十、已确认的设计决策

### 10.1 多检查汇总处理方式

**决策：合并显示**

同一学生在多次检查中违纪时，合并为一行显示：

| 序号 | 姓名 | 班级 | 违纪次数 | 累计扣分 | 违纪详情 |
|------|------|------|----------|----------|----------|
| 1 | 张三 | 软件1班 | 3 | 6分 | 迟到(2次)、未打扫(1次) |

### 10.2 通报记录保存

**决策：保存历史记录**

每次生成通报保存记录，支持：
- 查看历史通报列表
- 重新下载已生成的文件
- 基于历史通报重新生成

### 10.3 评级结果导出

**决策：支持导出为通报文档**

评级结果可以使用通报模板导出，例如：
- "12月第3周优秀班级通报"
- "本学期卫生班级汇总通报"

需要新增评级专用变量：
- `{{levelName}}` - 等级名称
- `{{ratingDate}}` - 评级日期
- `{{ratingPeriod}}` - 评级周期

### 10.4 Tab结构

**决策：5个核心Tab**

```
概览 | 检查管理 | 通报中心 | 评级管理 | 统计分析
```

其中"打分人员"并入"检查管理"Tab。

#### 自定义变量

用户可以定义自己的变量，存储在通报模板中：

```json
{
  "customVariables": [
    { "name": "schoolName", "label": "学校名称", "defaultValue": "XX职业技术学院" },
    { "name": "deptLeader", "label": "部门负责人", "defaultValue": "张主任" },
    { "name": "signature", "label": "落款单位", "defaultValue": "学生处" }
  ]
}
```

使用方式：`{{schoolName}}`、`{{deptLeader}}`、`{{signature}}`
