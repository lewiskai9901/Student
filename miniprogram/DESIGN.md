# 小程序量化管理功能设计方案

## 一、功能概述

小程序端与Web端共用同一套后端API，主要面向**班主任**和**检查员/管理员**角色，提供量化检查的移动端查看和申诉功能。

### 用户角色
| 角色 | 权限说明 |
|------|---------|
| 班主任 | 查看本班检查记录、扣分明细、提交申诉、查看申诉状态 |
| 检查员/管理员 | 查看所有检查计划、检查记录、审核申诉（暂不含打分功能） |

### 功能对照表（Web端 vs 小程序端）

| Web端功能 | 小程序端 | 说明 |
|-----------|---------|------|
| 检查计划列表 | ✅ | 查看计划列表和详情 |
| 检查计划创建/编辑 | ❌ | 暂不支持，需在Web端操作 |
| 检查计划状态管理 | ❌ | 暂不支持 |
| 检查记录列表 | ✅ | 查看检查记录列表 |
| 检查记录详情（管理员） | ✅ | 班级列表、排名视图 |
| 本班检查详情（班主任） | ✅ | 本班扣分详情 |
| 日常检查打分 | ❌ | 暂不开发 |
| 申诉管理-我的申诉 | ✅ | 查看和提交申诉 |
| 申诉管理-待我审核 | ✅ | 审核申诉 |
| 申诉管理-公示中 | ✅ | 查看公示申诉 |
| 申诉管理-全部申诉 | ✅ | 管理员查看 |
| 申诉统计分析 | ✅ | 图表展示 |
| 检查模板管理 | ❌ | 配置功能，Web端操作 |
| 加权/评级配置 | ❌ | 配置功能，Web端操作 |

│   └── index.vue
